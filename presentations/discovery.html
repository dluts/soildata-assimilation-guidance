Metadata & discovery

  • In order to combine data, you first have to find it, and understand usage constraints
  • For colleagues to find data on intranet (sharepoint), and externals via portals

Metadata harvests

  • Metadata of various sources is collected in central repositories to increase findability
  • Usually from regional up to country and europe
  • Vice versa is also of interest to facilitate colleagues to discover relevant data from external sources

Metadata standards

  • To facilitate wide discovery, communities should agree on adopting a metadata standard
  • Various communities have different metadata standards, central repositories can facilate transformations
  • Common standards are ISO19115, DublinCore/DCAT, Datacite, ...
    Metadata protocols
  • A metadata standard is often combined with a protocol for metadata exchange
  • ISO19115 - Catalogue Service for the Web, OGC API Records
  • DataCite - OAI-PMH
  • DCAT - SPARQL; CKAN-API

Search data via search engines

  • The schema.org ontology adds structured data to websites
  • Schema.org includes a topic related to [catalogues and datasets](https://schema.org/Dataset)
  • Google offers a [dataset search](https://datasetsearch.research.google.com/) based on schema.org/Dataset
  • Many catalogues (and data servers) now include schema.org annotations in their html representation of metadata records
  • UK government wrote a guidance document on how to optimally have your data found on search engines
  • Google and Yandex provide a [validator](https://search.google.com/test/rich-results/) to validate your annotations

Zenodo/Dataverse

GeoNetwork

  • Software widely used in geospatial domain, including INSPIRE GeoPortal (soon)
  • ISO19115 oriented, but supports other models via plugins

  • Participatory approach via github
  • Publication as iso19115 in pycsw

DCAT

  • Standard by W3C, adopted in Semantic web and open data domain
